# Support

## How to file issues and get help

SpecX Bot uses GitHub issues to track bugs and feature requests. Please search the existing issues before filing new issues to avoid duplicates. For new issues, file your bug or feature request as a new issue.

For help or questions about using SpecX Bot, please:

- Open a [GitHub issue](https://github.com/hjk1995/specx-bot/issues/new) for bug reports, feature requests, or questions
- Check the [README](./README.md) for comprehensive documentation on SpecX Bot features
- Review the [Spec-Driven Development guide](./spec-driven.md) for detailed documentation on the SDD methodology
- Consult the [original Spec Kit documentation](https://github.com/github/spec-kit) for SDD fundamentals

## Project Status

**SpecX Bot** is a fork of [GitHub Spec Kit](https://github.com/github/spec-kit) with enhanced role-based persona capabilities. This project is actively maintained and welcomes community contributions.

## Reporting Issues

When reporting issues, please include:

- Your operating system
- Python version (`python --version`)
- SpecX Bot version (`specx --version`)
- AI agent you're using (Cursor, Claude Code, Copilot, etc.)
- Development strategy (Role-Based or Traditional)
- Selected personas (if using Role-Based strategy)
- Steps to reproduce the issue
- Expected vs actual behavior
- Any relevant error messages or logs
